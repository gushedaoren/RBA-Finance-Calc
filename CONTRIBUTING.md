# Contributing to RBA-Finance-Calc

Thank you for your interest in contributing!

## How to Contribute

### 🐛 Report a Bug

Opening a GitHub Issue:

1. **Use the "Bug Report" template** (or create one with clear title)
2. **Describe the bug**: What happened vs. what you expected
3. **Include the reproduction steps**: Exact inputs, what result you get, what you expect
4. **Add a minimal code snippet** if applicable

### ✨ Propose a Feature

1. **Open a GitHub Issue first** to discuss the feature
2. Include **why** it would be useful and **what** it should do
3. If it's a **new financial calculation** — reference the formula and TI BA II Plus behavior

### 🌿 Pull Requests

1. Fork the repo and create a branch off `main`
2. Follow the existing code style (2-space indentation, descriptive variable names)
3. **Add tests** for any new functionality or bug fixes
4. Make sure all existing tests pass: `swift test`
5. Open a PR with a clear description

## Code Style Guidelines

- Use Swift's idiomatic naming conventions
- Document public APIs with Markdown-style comments
- Keep functions focused and testable
- All public types should include `public` visibility explicitly

## Questions?

Open an issue or reach out at [our App Store](https://apps.apple.com/app/id1545331477).
